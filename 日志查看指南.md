## 📌 注意事项

1. **日志文件大小**: PM2 会自动轮转日志，但建议定期清理旧日志
2. **敏感信息**: 日志中可能包含业务数据，注意保密
3. **性能影响**: 实时监控日志会占用一定网络带宽
4. **时区**: 日志时间为服务器时区（UTC+8）

## 🔄 日志轮转配置

PM2 默认配置：
- 日志文件按日期命名
- 自动创建新文件
- 旧文件保留在 `/root/.pm2/logs/` 目录

如需修改配置，可以在 PM2 启动时指定：
```bash
pm2 start dist/index.js --name fuxin --log-date-format "YYYY-MM-DD HH:mm:ss"
```

## 📞 问题排查流程

1. **用户报告问题** → 记录问题发生时间
2. **下载对应时间段日志** → 使用上述下载命令
3. **搜索关键词** → 使用 analyze-logs.sh 或 grep
4. **分析错误原因** → 查看前后日志上下文
5. **验证修复** → 部署后继续监控日志
