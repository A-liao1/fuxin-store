# 后端接口认证修复说明

## 问题描述
后端的所有业务接口在没有登录的情况下也可以正常访问，存在严重的安全隐患。

## 安全风险
- 任何人都可以直接访问 API 获取、修改、删除数据
- 没有用户身份验证和权限控制
- 数据完全暴露，无任何保护

## 修复方案

### 1. 创建 JWT 认证中间件
新建文件：`backend/src/middleware/auth.ts`

该中间件会：
- 从请求头中提取 JWT token
- 验证 token 的有效性
- 将用户信息附加到 request 对象
- 如果验证失败，返回 401 错误

### 2. 应用认证中间件到所有业务路由
修改文件：`backend/src/index.ts`

为以下路由添加认证保护：
- `/api/products` - 商品管理
- `/api/suppliers` - 供应商管理
- `/api/customers` - 客户管理
- `/api/purchases` - 采购管理
- `/api/sales` - 销售管理
- `/api/inventory` - 库存管理
- `/api/settlements` - 结算管理
- `/api/reports` - 报表查询
- `/api/dashboard` - 仪表盘数据
- `/api/brands` - 品牌管理

### 3. 不需要认证的路由
- `/api/auth/login` - 登录接口
- `/api/auth/captcha` - 验证码接口
- `/health` - 健康检查

## 测试步骤

### 1. 重新构建后端
```bash
cd backend
npm run build
```

### 2. 启动后端服务
```bash
npm start
# 或者使用开发模式
npm run dev
```

### 3. 测试未认证访问（应该返回 401）
```bash
# 测试获取商品列表（未登录）
curl http://localhost:3001/api/products

# 预期返回：
# {"success":false,"message":"未提供认证令牌"}
```

### 4. 测试已认证访问
```bash
# 1. 先登录获取 token
curl -X POST http://localhost:3001/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'

# 2. 使用返回的 token 访问接口
curl http://localhost:3001/api/products \
  -H "Authorization: Bearer YOUR_TOKEN_HERE"

# 预期返回：商品列表数据
```

### 5. 前端测试
1. 清除浏览器 localStorage（模拟未登录状态）
2. 直接访问 http://localhost:5173/dashboard
3. 应该自动跳转到登录页
4. 登录后可以正常访问所有页面

## 技术细节

### JWT Token 格式
```
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

### Token 有效期
- 24小时（在 `backend/src/routes/auth.ts` 中配置）

### 错误响应
```json
{
  "success": false,
  "message": "未提供认证令牌"
}
```
或
```json
{
  "success": false,
  "message": "认证令牌无效或已过期"
}
```

## 部署注意事项

1. 重新构建后端代码
2. 重新部署到服务器
3. 重启 PM2 进程
4. 确保前端的 token 存储和发送逻辑正常工作
