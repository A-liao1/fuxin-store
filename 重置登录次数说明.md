# 重置登录次数限制说明

## 快速重置方法

### 方法1：使用脚本（推荐）

运行交互式脚本：
```bash
./reset-login-attempts.sh
```

然后选择操作：
1. 查看所有登录尝试记录
2. 重置指定IP的登录尝试记录
3. 清除所有登录尝试记录
4. 退出

### 方法2：使用curl命令

#### 1. 查看所有登录尝试记录
```bash
curl http://localhost:3001/api/auth/login-attempts
```

返回示例：
```json
{
  "success": true,
  "total": 2,
  "records": [
    {
      "ip": "127.0.0.1",
      "attempts": 8,
      "status": "已锁定",
      "remainingTime": "25分钟",
      "lockedUntil": "2026-02-22 15:30:00"
    },
    {
      "ip": "192.168.1.100",
      "attempts": 3,
      "status": "正常",
      "remainingTime": "-",
      "lockedUntil": "-"
    }
  ]
}
```

#### 2. 重置指定IP的登录尝试记录
```bash
# 重置本地IP (127.0.0.1)
curl -X POST http://localhost:3001/api/auth/reset-login-attempts \
  -H "Content-Type: application/json" \
  -d '{"ip":"127.0.0.1"}'

# 重置其他IP
curl -X POST http://localhost:3001/api/auth/reset-login-attempts \
  -H "Content-Type: application/json" \
  -d '{"ip":"192.168.1.100"}'
```

#### 3. 清除所有登录尝试记录
```bash
curl -X POST http://localhost:3001/api/auth/reset-login-attempts \
  -H "Content-Type: application/json" \
  -d '{"ip":"all"}'
```

### 方法3：重启后端服务

由于登录尝试记录存储在内存中，重启后端服务会自动清除所有记录：

```bash
# 停止服务
# Ctrl+C 或者
pm2 stop fuxin

# 启动服务
npm start
# 或者
pm2 start dist/index.js --name fuxin
```

## API接口说明

### 1. 查看登录尝试记录

**接口**: `GET /api/auth/login-attempts`

**说明**: 查看所有IP的登录尝试记录和锁定状态

**返回字段**:
- `success`: 是否成功
- `total`: 记录总数
- `records`: 记录列表
  - `ip`: IP地址
  - `attempts`: 失败次数
  - `status`: 状态（正常/已锁定/锁定已过期）
  - `remainingTime`: 剩余锁定时间
  - `lockedUntil`: 锁定截止时间

### 2. 重置登录尝试记录

**接口**: `POST /api/auth/reset-login-attempts`

**参数**:
```json
{
  "ip": "127.0.0.1"  // 要重置的IP地址，或 "all" 清除所有
}
```

**返回**:
```json
{
  "success": true,
  "message": "已重置IP 127.0.0.1 的登录尝试记录"
}
```

## 常见问题

### Q1: 如何查看我的IP地址？
A: 查看后端日志，登录时会输出客户端IP：
```
客户端IP: 127.0.0.1
```

### Q2: 为什么重置后还是被锁定？
A: 可能的原因：
1. 重置的IP地址不正确
2. 浏览器缓存，尝试清除缓存或使用无痕模式
3. 后端服务未重启，确保使用最新代码

### Q3: 生产环境如何重置？
A: 生产环境建议：
1. 使用SSH连接到服务器
2. 运行curl命令重置
3. 或者添加管理后台界面

### Q4: 可以永久禁用某个IP吗？
A: 当前版本不支持，但可以通过以下方式实现：
1. 在Nginx层面配置IP黑名单
2. 修改代码添加黑名单功能
3. 使用防火墙规则

## 生产环境建议

### 1. 添加认证保护
重置接口应该添加管理员认证：
```typescript
router.post('/reset-login-attempts', authenticateToken, async (req, res) => {
  // 检查是否是管理员
  if (req.user?.role !== 'admin') {
    return res.status(403).json({ 
      success: false, 
      message: '权限不足' 
    });
  }
  // ... 重置逻辑
});
```

### 2. 记录操作日志
```typescript
console.log(`管理员 ${req.user?.username} 重置了IP ${ip} 的登录尝试记录`);
```

### 3. 使用Redis持久化
```typescript
// 使用Redis替代内存存储
const redis = new Redis();

router.post('/reset-login-attempts', async (req, res) => {
  const { ip } = req.body;
  
  if (ip === 'all') {
    await redis.del(await redis.keys('login_attempts:*'));
    await redis.del(await redis.keys('login_locked:*'));
  } else {
    await redis.del(`login_attempts:${ip}`);
    await redis.del(`login_locked:${ip}`);
  }
});
```
