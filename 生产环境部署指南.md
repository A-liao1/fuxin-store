# 生产环境部署指南

## 优化的部署脚本

### 新增功能

1. **数据库自动备份**
   - 部署前自动备份数据库
   - 保留最近10个备份
   - 备份文件带时间戳

2. **进程强制清理**
   - 自动停止PM2进程
   - 强制杀死残留进程
   - 清理端口占用

3. **健康检查**
   - 检查进程状态
   - 检查端口监听
   - 检查HTTP响应

4. **完整的错误处理**
   - 遇到错误立即停止
   - 显示详细错误信息
   - 彩色输出便于识别

## 使用方法

### 1. 生产环境部署

```bash
./deploy-production.sh
```

#### 部署流程

1. **确认部署** - 需要输入 `yes` 确认
2. **构建后端** - 编译TypeScript代码
3. **构建前端** - 打包React应用
4. **准备部署包** - 创建tar.gz压缩包
5. **上传到服务器** - 通过SCP上传
6. **服务器端部署**
   - 备份数据库
   - 停止服务（强制清理）
   - 解压新版本
   - 安装依赖
   - 启动服务
7. **健康检查** - 验证服务正常运行

#### 特性

- ✅ 自动数据库备份
- ✅ 强制进程清理
- ✅ 端口占用检查
- ✅ 健康检查
- ✅ 彩色输出
- ✅ 错误处理
- ✅ 部署历史保留

### 2. 数据库管理

```bash
./manage-database.sh
```

#### 功能菜单

1. **备份生产数据库到本地**
   - 下载生产数据库到本地
   - 自动创建备份目录
   - 显示文件大小

2. **查看服务器上的备份列表**
   - 列出所有备份文件
   - 显示文件大小和时间

3. **恢复数据库（从服务器备份）**
   - 选择备份文件
   - 自动停止服务
   - 恢复后重启服务

4. **上传本地数据库到服务器**
   - 上传本地数据库
   - 自动备份当前数据库
   - 替换生产数据库

5. **下载生产数据库到本地**
   - 下载到本地备份目录
   - 可选择替换本地开发数据库

6. **清理旧备份**
   - 清理服务器备份
   - 清理本地备份
   - 保留指定数量的最新备份

## 数据库备份策略

### 自动备份

部署脚本会在以下时机自动备份：

1. **部署前备份**
   - 位置: `/root/backups/database.db.YYYYMMDD_HHMMSS.bak`
   - 保留: 最近10个

2. **恢复前备份**
   - 位置: `/root/fuxin/database.db.before_restore.bak`
   - 保留: 最新1个

3. **上传前备份**
   - 位置: `/root/backups/database.db.YYYYMMDD_HHMMSS.before_upload.bak`
   - 保留: 所有

### 手动备份

```bash
# 使用管理脚本
./manage-database.sh
# 选择: 1. 备份生产数据库到本地

# 或使用命令
scp root@104.223.59.170:/root/fuxin/database.db ./database-backups/database.db.$(date +%Y%m%d_%H%M%S).bak
```

## 进程管理

### 停止服务的方式

部署脚本会按以下顺序停止服务：

1. **PM2停止**
   ```bash
   pm2 stop fuxin
   pm2 delete fuxin
   ```

2. **强制杀死进程**
   ```bash
   ps aux | grep "node.*dist/index.js" | grep -v grep | awk '{print $2}' | xargs kill -9
   ```

3. **清理端口占用**
   ```bash
   lsof -ti:3001 | xargs kill -9
   ```

### 常用PM2命令

```bash
# 查看状态
ssh root@104.223.59.170 'pm2 status'

# 查看日志
ssh root@104.223.59.170 'pm2 logs fuxin'

# 重启服务
ssh root@104.223.59.170 'pm2 restart fuxin'

# 停止服务
ssh root@104.223.59.170 'pm2 stop fuxin'

# 启动服务
ssh root@104.223.59.170 'pm2 start fuxin'
```

## 健康检查

部署完成后会自动执行以下检查：

1. **PM2状态检查**
   - 验证服务在PM2中运行

2. **进程检查**
   - 验证Node.js进程存在

3. **端口检查**
   - 验证端口3001正在监听

4. **HTTP检查**
   - 访问 `/health` 接口
   - 验证返回200状态码

如果任何检查失败，部署会报错并退出。

## 故障排查

### 问题1：部署失败，服务无法启动

**检查步骤：**

1. 查看PM2日志
   ```bash
   ssh root@104.223.59.170 'pm2 logs fuxin --lines 50'
   ```

2. 检查端口占用
   ```bash
   ssh root@104.223.59.170 'lsof -i:3001'
   ```

3. 检查进程
   ```bash
   ssh root@104.223.59.170 'ps aux | grep node'
   ```

4. 手动启动测试
   ```bash
   ssh root@104.223.59.170
   cd /root/fuxin
   node dist/index.js
   ```

### 问题2：数据库文件丢失

**恢复步骤：**

1. 查看备份
   ```bash
   ssh root@104.223.59.170 'ls -lh /root/backups/'
   ```

2. 使用管理脚本恢复
   ```bash
   ./manage-database.sh
   # 选择: 3. 恢复数据库
   ```

### 问题3：进程无法停止

**解决方法：**

```bash
ssh root@104.223.59.170

# 强制杀死所有相关进程
ps aux | grep -E "node.*fuxin|node.*dist/index.js" | grep -v grep | awk '{print $2}' | xargs kill -9

# 清理端口
lsof -ti:3001 | xargs kill -9

# 重新启动
cd /root/fuxin
pm2 start dist/index.js --name fuxin
```
