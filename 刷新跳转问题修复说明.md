# 刷新页面跳转登录页问题修复

## 问题描述
登录成功后，刷新页面会重新跳转到登录页面

## 根本原因
`AuthContext` 使用 `useEffect` 异步读取 localStorage，导致组件首次渲染时 `isAuthenticated` 为 `false`，触发了 `ProtectedRoute` 的重定向逻辑。

## 修复方案

### 1. 修改 AuthContext 初始化逻辑
将 localStorage 的读取从 `useEffect` 改为 `useState` 的初始化函数，确保同步读取：

```typescript
const [user, setUser] = useState<User | null>(() => {
  // 在初始化时同步读取 localStorage
  const storedUser = localStorage.getItem('user');
  const storedToken = localStorage.getItem('token');
  if (storedUser && storedToken) {
    try {
      return JSON.parse(storedUser);
    } catch (error) {
      console.error('Failed to parse stored user:', error);
      localStorage.removeItem('user');
      localStorage.removeItem('token');
      return null;
    }
  }
  return null;
});
```

### 2. 优化响应拦截器
避免在登录页面重复跳转：

```typescript
if (error.response && error.response.status === 401) {
  if (!window.location.pathname.includes('/login')) {
    localStorage.removeItem('token');
    localStorage.removeItem('user');
    window.location.href = '/login';
  }
}
```

## 测试步骤

1. 启动前端项目：`cd frontend && npm run dev`
2. 访问 http://localhost:5173
3. 使用管理员账号登录（admin/admin123）
4. 登录成功后，刷新页面（F5 或 Ctrl+R）
5. 确认页面保持在当前页面，不会跳转到登录页
