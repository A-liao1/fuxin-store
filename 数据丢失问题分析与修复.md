# 数据丢失问题分析与修复

## 🚨 严重问题
每次部署后，生产环境的所有数据都被清空，包括：
- 用户表（Users）
- 商品表（Products）
- 供应商表（Suppliers）
- 客户表（Customers）
- 采购记录（Purchases）
- 销售记录（Sales）
- 品牌表（Brands）

## 问题根源

### 1. 数据库同步配置错误
在 `backend/src/index.ts` 中使用了：
```typescript
await sequelize.sync({ alter: true });
```

**问题说明：**
- `sync({ alter: true })` 会在每次启动时修改数据库表结构
- 在某些情况下（特别是模型定义变化时），可能会删除并重建表
- 这会导致所有数据丢失
- **这个配置绝对不应该在生产环境使用！**

### 2. 没有区分开发和生产环境
- 代码没有检查 `NODE_ENV` 环境变量
- 开发和生产使用相同的数据库同步策略
- 生产环境应该只连接数据库，不应该修改表结构

### 2. 修改数据库同步逻辑 ✅
修改 `backend/src/index.ts`：
```typescript
// 修改前（危险！）
await sequelize.sync({ alter: true });

// 修改后（安全）
const isProduction = process.env.NODE_ENV === 'production';
if (isProduction) {
  // 生产环境：只检查连接，不修改表结构
  await sequelize.authenticate();
  console.log('数据库连接成功（生产模式：不修改表结构）');
} else {
  // 开发环境：可以自动同步表结构
  await sequelize.sync({ alter: true });
  console.log('数据库模型同步完成（开发模式）');
}
```

### 3. 修改部署脚本 ✅
在 `deploy-production.sh` 中设置环境变量：
```bash
# 修改前
pm2 start dist/index.js --name $APP_NAME --time

# 修改后
NODE_ENV=production pm2 start dist/index.js --name $APP_NAME --time --env production
```

### 4. 创建紧急恢复脚本 ✅
创建了 `emergency-restore.sh` 脚本，用于快速恢复数据：
```bash
./emergency-restore.sh
```

功能：
- 列出所有可用备份
- 显示每个备份的数据量
- 交互式选择要恢复的备份
- 自动停止服务、恢复数据、重启服务
