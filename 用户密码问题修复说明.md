# 用户密码问题修复说明

## 问题描述
每次重新部署后，登录账号的密码总是错误，无法登录系统。

## 问题原因
1. 生产环境数据库中没有用户数据
2. 部署脚本没有包含用户初始化步骤
3. 虽然数据库文件被保留，但Users表是空的

## 解决方案

### 1. 创建用户初始化脚本
创建了 `init-production-users.js` 脚本，用于：
- 检查并创建管理员账户（admin/admin123）
- 检查并创建用户账户（liuguozhi/lgzlww123）
- 验证密码是否正确
- 如果密码不正确，自动重置

### 2. 修改部署脚本
修改了 `deploy-production.sh`，添加了：
- 在准备部署包时复制用户初始化脚本
- 在启动服务前自动运行用户初始化脚本

### 3. 手动初始化用户（已完成）
在生产环境上手动运行了初始化脚本：
```bash
cd /root/fuxin
node init-production-users.js
```

结果：
```
✓ 管理员账户创建成功
✓ 用户 liuguozhi 创建成功

当前所有用户:
- admin (系统管理员) - 角色: admin
- liuguozhi (刘国治) - 角色: admin
```

## 后续部署
从现在开始，每次运行 `./deploy-production.sh` 部署时，会自动：
1. 上传用户初始化脚本
2. 在启动服务前运行初始化脚本
3. 确保用户账户存在且密码正确

## 手动重置密码
如果需要手动重置密码，可以在服务器上运行：
```bash
ssh root@104.223.59.170
cd /root/fuxin
node init-production-users.js
```

## 注意事项
1. 用户初始化脚本会检查用户是否存在
2. 如果用户已存在，会验证密码是否正确
3. 如果密码不正确，会自动重置为默认密码
4. 不会删除或覆盖现有用户数据
