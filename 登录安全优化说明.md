# 登录安全优化说明

## 优化内容

1. 去掉验证码登录
2. 增加登录错误次数验证（基于IP地址）

### 2. 登录错误次数限制

#### 限制规则
- 同一IP地址连续登录错误最多8次
- 超过8次后，该IP将被锁定30分钟
- 锁定期间无法访问登录接口
- 30分钟后自动解锁
- 登录成功后，清除该IP的错误记录

#### 实现方式
- 使用内存存储登录尝试记录（Map结构）
- 记录每个IP的失败次数和锁定时间
- 每小时自动清理过期的锁定记录

#### 提示信息
- 登录失败时：显示剩余尝试次数（如"还剩5次尝试机会"）
- 达到8次后：提示"登录失败次数过多，账号已被锁定30分钟"
- 锁定期间：提示"登录失败次数过多，请在X分钟后重试"

```typescript
const getClientIp = (req: express.Request): string => {
  const forwarded = req.headers['x-forwarded-for'];
  if (typeof forwarded === 'string') {

  }
  return req.ip || req.connection.remoteAddress || 'unknown';
```

```typescript
const recordFailedLogin = (ip: string): void => {
  const record = loginAttempts.get(ip) || { attempts: 0, lockedUntil: null };
  record.attempts += 1;
  
  // 达到8次，锁定30分钟
  if (record.attempts >= 8) {
    const lockTime = new Date();
    lockTime.setMinutes(lockTime.getMinutes() + 30);
    record.lockedUntil = lockTime;
  }
  
  loginAttempts.set(ip, record);
};
```

```typescript
const clearLoginAttempts = (ip: string): void => {
  loginAttempts.delete(ip);
```

## 安全特性

### 优点
1. 防止暴力破解：限制同一IP的登录尝试次数
2. 自动解锁：30分钟后自动恢复，无需人工干预
3. 友好提示：显示剩余尝试次数和解锁时间
4. 内存存储：快速响应，不增加数据库负担
5. 自动清理：定期清理过期记录，避免内存泄漏

### 注意事项
1. IP获取：支持反向代理（nginx）的 X-Forwarded-For 头
2. 内存存储：服务重启后记录会丢失（可考虑使用Redis持久化）
3. 分布式部署：如果有多个后端实例，需要使用Redis等共享存储
4. IPv6支持：代码已支持IPv6地址

## 部署注意事项

### 1. Nginx配置
确保Nginx正确转发客户端IP：
```nginx
location /api {
    proxy_pass http://localhost:3001;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Real-IP $remote_addr;
}
```

### 2. 生产环境优化
考虑使用Redis存储登录尝试记录：
```typescript
// 使用Redis替代内存存储
import Redis from 'ioredis';
const redis = new Redis();

const recordFailedLogin = async (ip: string) => {
  const key = `login_attempts:${ip}`;
  const attempts = await redis.incr(key);
  
  if (attempts === 1) {
    await redis.expire(key, 1800); // 30分钟过期
  }
  
  if (attempts >= 8) {
    await redis.set(`login_locked:${ip}`, '1', 'EX', 1800);
  }
};
```
