# 登录错误限制修复说明

## 问题描述
之前部署的代码中，登录错误限制功能存在一个问题：
- 当用户名不存在时，只返回"用户名或密码错误"，不显示剩余尝试次数
- 只有当用户名存在但密码错误时，才会显示剩余次数

## 修复内容
修改了 `backend/src/routes/auth.ts` 文件中的登录逻辑：

```typescript
// 修复前
if (!user) {
  console.error('用户不存在:', username);
  recordFailedLogin(clientIp);
  return res.json({ success: false, message: '用户名或密码错误' });
}

// 修复后
if (!user) {
  console.error('用户不存在:', username);
  recordFailedLogin(clientIp);
  const newAttemptCheck = checkLoginAttempts(clientIp);
  const remainingAttempts = 8 - newAttemptCheck.attempts;
  
  if (remainingAttempts > 0) {
    return res.json({ 
      success: false, 
      message: `用户名或密码错误，还剩${remainingAttempts}次尝试机会` 
    });
  } else {
    return res.json({ 
      success: false, 
      message: '登录失败次数过多，账号已被锁定30分钟' 
    });
  }
}
```
